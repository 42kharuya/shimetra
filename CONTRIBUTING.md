# Contributing

このリポジトリはテンプレートとして使われる前提です。プロジェクト固有の手順は派生リポジトリ側で更新してください。

## 進め方

1. Issue を作成し、目的・範囲・受け入れ条件（Acceptance Criteria）を明確にする
2. 小さなPRに分割して提出する
3. レビュー指摘は「理由」「代替案」「影響範囲」をセットで返す

## PR ルール

- 変更理由が分かるタイトル/説明を書く
- 影響範囲（UI/API/DB/依存関係/設定）を列挙する
- 破壊的変更があれば明記する
- 必要に応じてスクリーンショットやログなどの証跡を添付する

## コーディング規約

派生先の技術スタックに合わせて `docs/development.md` を更新してください。
