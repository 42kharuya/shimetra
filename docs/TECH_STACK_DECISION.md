# Tech Stack Decision（初学者向け）

目的: 「迷って止まる」を防ぐため、MVPを 2〜4 週間で出す前提で **枯れた選択**を決めます。
このテンプレはドキュメント中心なので、派生リポジトリ側で最終決定してください。

## 1) まず決める（必須）

- ホスティング（Webをどこで動かすか）:
- データ保存（DB/ストレージ）:
- 認証（必要なら）:
- 決済（必要なら）:
- 計測（Analytics）:
- エラー監視（任意）:

## 2) 推奨デフォルト（迷ったらこれ）

- 言語: TypeScript（フルスタックならTypeScriptで統一）
- ホスティング: マネージド（デプロイが簡単なもの）
- DB: マネージドPostgres（将来の拡張が楽）
- 認証: まずは「メール」or「OAuth 1種類」だけ
- 決済: MVPでは「課金導線を最短」で置けるもの（サブスク/買い切りはPRDに合わせる）
- 計測: `signup` / `activation` / `purchase` だけ先に入れる（[docs/ANALYTICS_SPEC_TEMPLATE.md](ANALYTICS_SPEC_TEMPLATE.md)）
- 監視: エラー監視 + 簡単な死活監視

## 3) 判断の基準（3つだけ）

1. 1人で運用できるか（設定・監視が重くない）
2. ロールバックできるか（戻せない選択を避ける）
3. データを守れるか（バックアップ/復旧/削除依頼の運用が可能）

## 4) 決定事項の置き場

決めた内容は [docs/architecture.md](architecture.md) のADRに残してください（例: ADR 0001）。
