# Security Policy

## 脆弱性の報告

公開 Issue ではなく、メンテナに非公開で連絡してください。

- 連絡先: kharuya.0411.k@gmail.com
- 期待する情報:
  - 再現手順
  - 影響範囲
  - PoC（可能なら）
  - 環境情報（OS/ブラウザ/コミットなど）

可能であれば、GitHub の Security Advisory 機能（Private vulnerability reporting）も利用してください（派生リポジトリ側で有効化）。

## 対応方針

| 重要度   | 基準                                   | 目標対応時間        |
| -------- | -------------------------------------- | ------------------- |
| Critical | 認証バイパス・全データ漏洩・課金改ざん | 24 時間以内に緩和策 |
| High     | 一部ユーザーへの影響・権限昇格         | 72 時間以内         |
| Medium   | 軽微な情報漏洩・DoS                    | 7 日以内            |
| Low      | ベストプラクティス違反・設定ミス       | 計画対応            |

- 受領連絡: 2 営業日以内
- 修正公開: 可能な限り迅速に（必要なら段階的に緩和策 → 恒久対策）

## シークレット管理ルール（開発者向け）

- API キー・シークレットは `.env` に記述し、`.gitignore` で除外する
- `.env.example` に変数名のみ記載する（値は絶対にコミットしない）
- 本番シークレットは `wrangler secret put <VAR_NAME>` で登録し、リポジトリにコミットしない
- シークレットが漏洩した場合は即座にローテーション（手順: [docs/RUNBOOK.md](docs/RUNBOOK.md) §2）

## 公開について

修正が完了するまで詳細の公開は控えてください（利用者保護のため）。
CVE 番号の取得が必要な場合はメンテナに相談してください。
