# Security Policy

## 脆弱性の報告

公開Issueではなく、メンテナに非公開で連絡してください。

- 連絡先: （ここにメールアドレス or 連絡フォームURLを記載）
- 期待する情報:
  - 再現手順
  - 影響範囲
  - PoC（可能なら）
  - 環境情報（OS/ブラウザ/コミットなど）

可能であれば、GitHubのSecurity Advisory機能（Private vulnerability reporting）も利用してください（派生リポジトリ側で有効化）。

## 対応方針（例）

- 受領連絡: 数営業日以内
- 重要度評価: 影響範囲と再現性で判断
- 修正公開: 可能な限り迅速に（必要なら段階的に緩和策→恒久対策）

## 公開について

修正が完了するまで詳細の公開は控えてください（利用者保護のため）。
